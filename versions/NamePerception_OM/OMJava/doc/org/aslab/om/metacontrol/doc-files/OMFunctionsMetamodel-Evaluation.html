Evaluation Models

Following the evaluation model for each of the Function metamodel is schematically explained
relevance is the attribute for all of this elements related to its evaluation

the relevance of a Root Objective is given by the initial state of the functional model (domain control system designer decision)


for function and function specifications
confidence = 1-failure_probability


Objective

relevance = max(set of owners fg relevances), that is, the bigger relevance in between those of the function groundings that require the objective


FunctionGrounding

- if in ERROR -> relevance =0
- weighted sum of the relevances for all the objectives it is realising
     the weight is the relative relevance of the Fspec of this fg for the function of each objective



Function

failure probability = 1* product for all REALISABLE(1-Fspec.confidence)



FunctionSpecification

confidence of an Fspec is 1-failure_probability, and is static given knowledge


relevance of an FSpec is relative to to a certain F (function) (obvious, in our model one FSpec realises only one FSpec)
it is the probability of F not being achieved when FSpec is not available, normalized

double aux = 0;
